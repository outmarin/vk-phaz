# VK Phaz

Клиент VK в стиле iOS-Telegram: только чаты и вход/выход по `access_token`.
Нативный SwiftUI, ноль сторонних зависимостей.

## Как получить .ipa
1. Залей папку в GitHub-репозиторий.
2. Открой вкладку **Actions** → workflow **Build IPA** запустится сам (или жми *Run workflow*).
3. Скачай артефакт **VKPhaz-ipa** → внутри `VKPhaz.ipa`.

`.ipa` **неподписанный**. Поставить на устройство:
- **AltStore** / **Sideloadly** (подпишут бесплатным Apple ID, переустановка раз в 7 дней), либо
- свой сертификат Apple Developer.

## Вход
В приложении вставь `access_token` аккаунта VK. Токен со scope `messages`
получают отдельно (например через официальный OAuth VK / app_id мобильного клиента) —
приложение его только хранит и использует.

## Что сделано по безопасности
- Токен лежит в **Keychain** (`ThisDeviceOnly`), не в UserDefaults, не в коде.
- Секреты уходят в **теле POST**, не в URL (не попадают в логи/кэш).
- **ATS**: plaintext-HTTP запрещён — только TLS к `api.vk.com`.
- Приложение ходит **только** на VK API и грузит аватарки с CDN VK. Нет аналитики/SDK.
- Сообщения рендерятся как **обычный текст** (`Text`), не HTML — инъекции со стороны API невозможны.
- Читаются только нужные поля JSON — «лишнее» из ответа VK игнорируется.

## Ограничения (честно)
- Доступ к сообщениям через пользовательский токен — против ToS VK и может отвалиться/забанить аккаунт. Это на твой риск.
- Нет: медиа, вложений, звонков, уведомлений, редактирования. Только текстовые чаты, лента диалогов, отправка.
- В групповых беседах «своё/чужое» определяется грубо (см. `ponytail:` в `ChatView.swift`).
