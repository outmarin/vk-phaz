# TK notifier — уведомления при закрытом приложении

Маленький процесс, который держит VK Long Poll и шлёт новые входящие в твой Telegram-бот.
Пока он крутится где-то (ПК, Raspberry Pi, VPS, облако) — уведомления приходят **даже когда
приложение TK закрыто или телефон спит**. Само приложение для этого не нужно.

## Что нужно
- `VK_TOKEN` — тот же access_token, что в приложении.
- `TG_BOT_TOKEN` — токен бота от @BotFather.
- `TG_CHAT_ID` — твой chat_id (узнать у @userinfobot; сначала напиши `/start` своему боту).

## Запуск локально (проще всего)
```bash
VK_TOKEN=xxx TG_BOT_TOKEN=yyy TG_CHAT_ID=123456 python3 notify.py
```
Только Python 3, без зависимостей. Пусть окно висит — пока висит, уведомления идут.

## Docker
```bash
docker build -t tk-notifier .
docker run -d --restart=always \
  -e VK_TOKEN=xxx -e TG_BOT_TOKEN=yyy -e TG_CHAT_ID=123456 tk-notifier
```

## Облако (чтобы не держать ПК включённым)
Подойдёт любой хост с постоянным процессом (long-poll — это долгое соединение, не «спящий» веб-сервис):
- **Fly.io** / **Railway** / **VPS** — задеплой Docker-образ, задай 3 переменные окружения.
- На бесплатных «спящих» веб-сервисах (Render free) работать надёжно не будет — процесс должен жить постоянно.

## Честно
- Токен VK лежит на том хосте, где крутится процесс — держи это в секрете.
- Через пользовательский токен читать сообщения — против ToS VK, риск бана.
